## Why ReactFlow?

ReactFlow offers several advantages over Mermaid:

- **Performance**: Renders efficiently with thousands of nodes
- **Interactivity**: Users can pan, zoom, and manipulate the diagram
- **Accessibility**: Built with semantic HTML and ARIA labels
- **Customization**: Full control over styling and behavior
- **React Integration**: First-class support for React components within diagrams

<KeyTakeaway variant="insight" title="ReactFlow vs Mermaid">
  ReactFlow provides better performance for complex diagrams (handles thousands of nodes), built-in
  accessibility with ARIA labels, and seamless React integration. Unlike Mermaid's static SVG
  output, ReactFlow diagrams are fully interactive with pan, zoom, and keyboard navigation support.
</KeyTakeaway>

## Customization

All diagram presets accept the same props for customization:

```tsx
interface DiagramPresetProps {
  showControls?: boolean; // Show pan/zoom controls (default: true)
  height?: number | string; // Diagram height (default: varies)
  className?: string; // Additional CSS classes
  caption?: string; // Caption text displayed below the diagram
}
```

**Examples:**

```tsx
// Full height diagram with controls and caption
<MCPArchitecture
  height="100%"
  showControls={true}
  caption="The three-tier MCP architecture"
/>

// Compact diagram with custom caption
<AuthenticationFlow
  height={300}
  showControls={false}
  caption="User login flow showing valid and invalid paths"
/>

// With custom styling and caption
<PipelineFlow
  height={500}
  className="rounded-lg border border-border"
  caption="Six-stage MDX to HTML transformation pipeline"
/>
```

## Creating Custom Diagrams

If you need a custom diagram, use the base <GlossaryTooltip term="InteractiveDiagram" definition="The foundational ReactFlow wrapper component that provides pan/zoom controls, accessibility features, and consistent diagram styling across all preset diagrams">InteractiveDiagram</GlossaryTooltip> component directly:

```tsx
import { InteractiveDiagram } from '@/components/common/interactive-diagram';
import { type Node, type Edge } from '@xyflow/react';

const nodes: Node[] = [
  { id: '1', position: { x: 0, y: 0 }, data: { label: 'Start' } },
  { id: '2', position: { x: 250, y: 0 }, data: { label: 'Process' } },
  { id: '3', position: { x: 500, y: 0 }, data: { label: 'End' } },
];

const edges: Edge[] = [
  { id: 'e1-2', source: '1', target: '2' },
  { id: 'e2-3', source: '2', target: '3' },
];

export function CustomDiagram() {
  return <InteractiveDiagram nodes={nodes} edges={edges} title="My Custom Flow" height={400} />;
}
```

## Performance Notes

- Diagrams are <GlossaryTooltip term="Client-side rendering" definition="JavaScript-based rendering that executes in the browser after the initial HTML loads, enabling interactive features like pan and zoom in ReactFlow diagrams">**client-side rendered**</GlossaryTooltip> using React hooks
- Use `showControls={false}` for read-only diagrams to reduce bundle impact
- Each diagram is lightweight (< 5KB gzipped)
- Animations are <GlossaryTooltip term="GPU-accelerated" definition="Hardware-accelerated rendering that uses the graphics processing unit instead of CPU for smooth 60fps animations with CSS transforms and transitions">GPU-accelerated</GlossaryTooltip> for smooth performance

<KeyTakeaway variant="tip" title="Optimal Performance Settings">
  Each diagram weighs less than 5KB gzipped. For static content, use showControls={false} to
  eliminate pan/zoom controls and reduce the JavaScript bundle. All animations leverage GPU
  acceleration for consistent 60fps rendering.
</KeyTakeaway>
